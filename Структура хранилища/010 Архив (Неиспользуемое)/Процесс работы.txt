02. Архитектрура фабрики. Структура обработки данных

1. Raw - сырые файлы как есть                               |     
2. Extraction - достает текст и таблицы                     |
3. Chunking - режет на умные куски                          |  --> настраивается один раз. Меняется только придобавление новой инфы 
4. Embendding - кусок -> вектор по смыслу                   |
5. Vector Index - хранит и ищет похожие                     |
6. Graph Builder - строит связи сущностей                   |

7. Memory Store - помнит контекст агентов                   | 
8. Retrieval - достает нужное. точность поиска              |
9. Ranking - лучшее наверх. Сортирует куски. Релевантности  |  --> С этим ведется работа на постоянной основе
10.Synthesis - Собирает связный ответ                       | 
11.Agent Router - Направляет агенту                         |

03. Knowledge MCP

Универсальный разъем. Один протокол доступа для всех агентов. Каждый агент в собственной оболочеке Docker и обращается к единой БД базе знаний.

Любой агент:

- Разработки   |                                         |       - Вектор QDrant
- Аналитики    |   -->  Knowledge MCP (единый разъем)    |  -->  - Граф Neo4j
- Безопасноти  |                                         |       - Память MemO 


Функции ядра:

- semantic_search  (поиск по смыслу в векторах)
- hybrid_search (вектор + ключевые слова)
- graph_search (обход связей по графу)

06. Развернутая архитектура:

Девять принципов:

- AI-First
- Retrieval-First
- Cloud-Agnostic
- Agent-First
- Memory-First
- Docker
- Graph-First
- Self-hosted
- K8s-Ready


Docker- композиция:

- Ingest - Приемник, режет файлы на куски
- Vector-DB - QDrant - хранит векторы, ищет ближайших
- Graph-DB - Neo4j - сущности исвязи между ними
- Memory - MemO - что было, что знаю
- Router - Knowledge MCP - шлюз раздает знания

Агенты подключаются по MCP

Единый протокол доступа к базе знаний

База сама объясняте агентам как ей пользоваться:

- Пеерчень документов - что вообще есть в базе
- Как пользоваться - как правильно справшивать
- Синтез - собрать готовый ответ
- Инвентарь - список инструментов
- Функциональный канал - вызов функций, поиск, граф, память
