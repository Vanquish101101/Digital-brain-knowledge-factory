# 🚦 ПЕРЕД СТАРТОМ — памятка для запуска Фабрики знаний

> Скорми этот файл агенту **вместе с `core.md` и `rules.md`** ИЛИ пройди пункты сам.
> Цель — запустить Knowledge Factory с первого раза, без типовых проблем (забитый
> диск, нехватка RAM, своп, OOM, конфликты портов). Подходит для **Windows, macOS, Linux**.

---

## 1. ЧТО ТЕБЕ ПОНАДОБИТСЯ
- **Docker** (Desktop/OrbStack/Engine) + `docker compose`.
- **Отдельный диск или папка с запасом** ≥ 50 GB под данные (внешний SSD / не системный раздел).
- **Системный диск** ≥ 10 GB свободно под образы Docker.
- Вариант LLM (выбери один в п.4): локальный (Ollama) **или** через OpenRouter-ключ.

---

## 2. ВЫБЕРИ ПРОФИЛЬ ПО СВОЕЙ ПАМЯТИ (RAM)
Это самое важное решение — от него зависит, влезет ли всё в память.

### 🟢 LIGHT — 8–16 GB RAM (рекомендуется большинству)
Стек: **Qdrant + PostgreSQL/pgvector + MinIO + Redis + Kuzu** (граф без JVM).
- Эмбеддинги: **fastembed** (локально, на CPU, поддерживает русский) — без ключа.
- Reranker: **bge-reranker-base** локально (или пропустить на старте).
- Потребление ~4–8 GB. **БЕЗ OpenSearch, БЕЗ Neo4j** — они едят слишком много.

### 🔴 FULL — 32 GB+ RAM
+ OpenSearch (BM25) + Neo4j + локальные тяжёлые модели. Потребление ~16–24 GB.

> Правило: на < 32 GB **не включай OpenSearch и Neo4j** — это главные пожиратели памяти (JVM).

---

## 3. КУДА КЛАСТЬ ДАННЫЕ (чтобы не забить системный диск)
Тома данных размещай на **отдельном/внешнем диске**, не на системном.
Создай папку и bind-mount туда все тома:
- **macOS/Linux:** `/<диск>/knowledge-factory/data/{minio,qdrant,postgres,graph,redis}`
- **Windows:** `D:\knowledge-factory\data\...` (не на забитом `C:`)

---

## 4. LLM — выбери один вариант (нужен для графа и синтеза ответа)
LLM извлекает сущности (граф) и собирает связный ответ. Два пути:

### Вариант A — полностью локально (без ключей, приватно)
- **Ollama** + `qwen2.5:3b` (~2.5 GB, влезает в 16 GB) или `qwen2.5:7b` (~5–6 GB, нужно больше RAM).
- Плюс: данные не покидают компьютер. Минус: ест RAM.

### Вариант B — через OpenRouter (легко, не ест RAM)
- В стеке **LiteLLM** направляешь на OpenRouter, модель напр. `deepseek/deepseek-v4-flash` (дёшево).
- Плюс: 0 GB локально, качество выше. Минус: куски текста уходят в облако (не для чувствительных данных).

> Эмбеддинги в обоих вариантах — **локальные (fastembed)**. OpenRouter их не отдаёт.

---

## 5. PRE-FLIGHT — ПРОВЕРКИ ПЕРЕД `docker compose up`
Выполни и убедись, что всё ок:

**RAM и диск:**
- macOS/Linux: `free -h` (или `vm_stat`) · `df -h`
- Windows PowerShell: `systeminfo | findstr Memory` · `Get-PSDrive C`

**Docker запущен:** `docker info` (демон отвечает) · `docker compose version`

**Память Docker поднята** не ниже профиля (Settings → Resources на Mac/Win;
на Windows ещё `.wslconfig` → `[wsl2] memory=12GB`), оставь ОС ≥ 4 GB.

**Порты свободны** (иначе конфликт с другими контейнерами):
`6333` Qdrant · `5432` Postgres · `9000/9001` MinIO · `6379` Redis · `7474/7687` Neo4j · `9200` OpenSearch
- macOS/Linux: `lsof -i :ПОРТ` · Windows: `netstat -ano | findstr :ПОРТ`
- Если порт занят — поменяй host-порт в compose (напр. Postgres `5432 → 5433`).

**Освободи место при нехватке (безопасно):**
`docker builder prune -af` · `docker image prune -f`
(чужие данные/тома/контейнеры не удаляй без согласия владельца).

---

## 6. ЧТО ИНДЕКСИРОВАТЬ (scope)
Индексируй **только текст**. Исключи из обработки:
- видео / аудио / большие медиа, бинарники
- `node_modules`, `.git`, `venv`, `dist`, `build`, кэши
- архивы целиком
Иначе индексация будет долгой и забьёт диск бесполезными эмбеддингами.

---

## 7. ⚠️ ЧАСТЫЕ ОШИБКИ (НЕ ПОВТОРЯЙ)
| Симптом | Причина | Решение |
|---|---|---|
| `pull` образов падает, «no space left» | системный диск забит | LIGHT-профиль + тома на внешний диск + `prune` |
| контейнеры падают / всё тормозит, своп растёт | мало RAM (включили OpenSearch/Neo4j на 16 GB) | LIGHT-профиль |
| контейнер убивается (OOM / 137) | не поднят RAM-лимит Docker (Win WSL2 / Mac) | задать лимит ≥ профиля |
| `port is already allocated` | порт занят другим контейнером | сменить host-порт в compose |
| индексация идёт часами | в scope попали видео/код/архивы | оставить только текст |
| тома растут, диск C: переполняется | тома на системном диске | bind-mount на отдельный диск |

---

## 8. СТАРТ (только после пунктов 1–7)
```
docker compose up -d
```
1. Дождись **healthcheck** всех сервисов (`docker compose ps`).
2. Запусти индексацию выбранного scope (п.6).
3. Проверь **Knowledge MCP**: `semantic_search`, `hybrid_search`, `graph_search`.
4. Готово — Фабрика онлайн. ✅
