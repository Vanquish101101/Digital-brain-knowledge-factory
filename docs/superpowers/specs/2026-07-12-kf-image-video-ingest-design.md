# Расширение kf.py: индексация картинок и видео

Дата: 2026-07-12
Статус: одобрено пользователем, готово к написанию implementation-плана

## Контекст и цель

`kf.py` (пайплайн приёма знаний Digital Brain) сейчас индексирует только текстовые
форматы (`.md .txt .csv .html .docx .pdf`) — см. `kf/scope.py` и `kf/extract.py`.
Пользователь хочет так же автоматически индексировать картинки (в основном —
скриншоты и документы с текстом) и видео (важны и речь, и визуальный ряд
примерно поровну), чтобы после однократного разбора эта информация была
постоянно доступна через `semantic_search`/`ask`, без необходимости повторно
подключать источник.

Ключевой архитектурный факт, на котором строится решение: весь пайплайн после
`extract_text()` (chunk → embed → Qdrant/Postgres/MinIO) уже работает с любым
текстом одинаково. Задача сводится к тому, чтобы получить из картинки/видео
текстовое представление — дальше всё пойдёт по существующим рельсам.

## Требования (из брейнсторминга с пользователем)

- Картинки: приоритет — OCR (скриншоты/документы с текстом).
- Видео: важны и транскрипт речи, и содержимое кадров — примерно поровну.
- Запуск полностью автоматический через `kf.py ingest`, без участия агента
  в моменте разбора.
- Подход — гибрид: локальная обработка как основной путь (дёшево, приватно,
  соответствует уже принятому в проекте принципу «эмбеддинги локально, LLM —
  через OpenRouter»), с добором через OpenRouter vision-модель только когда
  локального OCR недостаточно (мало текста → скорее фото/сцена).

## Архитектура: новые модули

Одна ответственность на модуль (не расширяем `extract.py` вслепую):

| Модуль | Функция | Назначение |
|---|---|---|
| `kf/ocr.py` | `extract_text_from_image(path) -> str` | Tesseract OCR, языки `rus+eng` |
| `kf/vision_caption.py` | `caption_image(settings, path) -> str` | HTTP-вызов к OpenRouter vision-модели (по образцу `kf/llm.py`, но с изображением в теле запроса) |
| `kf/video.py` | `extract_audio(path) -> Path`, `sample_frames(path, interval_seconds) -> list[Path]` | Оркестрация через ffmpeg + очистка временных файлов |
| `kf/transcribe.py` | `transcribe_audio(path) -> str` | Обёртка над `faster-whisper`, модель кэшируется в `MODEL_CACHE_DIR` (как fastembed) |

`kf/extract.py` дополняется диспетчеризацией по расширению файла (см. «Поток
данных» ниже) и делегирует в модули выше.

**Breaking-изменение сигнатуры:** `extract_text(path)` → `extract_text(path, settings)`.
Картинкам/видео нужен доступ к `Settings` (ключ OpenRouter, имя vision-модели,
размер whisper-модели, порог OCR, интервал сэмплирования кадров). Затрагивает
`kf/ingest.py` (вызывающая сторона) и существующие тесты `test_extract.py`,
`test_ingest.py` — ожидаемо при TDD RED-GREEN, не считается регрессией.

## Поток данных

**Картинка** (`.png .jpg .jpeg .webp`):
```
файл → Tesseract OCR → текст
  если len(текст.strip()) < IMAGE_CAPTION_THRESHOLD_CHARS
    → OpenRouter vision-модель → описание сцены
  → итоговый текст = OCR-текст (+ описание, если было)
  → chunk_text → embed → Qdrant/Postgres/MinIO (как обычно)
```

**Видео** (`.mp4 .mov .mkv .webm`):
```
файл → ffmpeg извлекает аудиодорожку (temp .wav)
     → faster-whisper → транскрипт речи
файл → ffmpeg сэмплирует кадры (раз в VIDEO_FRAME_INTERVAL_SECONDS)
     → каждый кадр проходит ту же цепочку OCR→caption, что и картинка
     → результат помечается таймкодом кадра
итоговый текст = "[Транскрипт]\n...\n\n[Кадр 00:00:15]\n...\n\n[Кадр 00:00:30]\n..."
temp-файлы (wav, кадры) удаляются после обработки
     → chunk_text → embed → ... (как обычно)
```

Оригинальный файл (картинка/видео) по-прежнему целиком грузится в MinIO — так
же, как сейчас происходит с `.pdf`/`.docx`.

## Конфигурация

Новые переменные в `.env` / `kf/config.py`:

```
OCR_LANGUAGES=rus+eng
IMAGE_CAPTION_THRESHOLD_CHARS=20
VISION_MODEL=google/gemini-2.5-flash
VIDEO_FRAME_INTERVAL_SECONDS=15
WHISPER_MODEL_SIZE=small
```

Новые Python-зависимости (`pyproject.toml`): `pytesseract`, `Pillow`, `faster-whisper`.

Новые расширения в `kf/scope.py` → `INCLUDED_EXTENSIONS`:
`.png .jpg .jpeg .webp` (картинки), `.mp4 .mov .mkv .webm` (видео).

**Системные зависимости (ручная установка, не через pip):**
- **Tesseract OCR** — Windows-инсталлятор + языковой пакет `rus`. Без него
  `kf/ocr.py` не заработает.
- **ffmpeg** — бинарник в `PATH`, нужен для извлечения звука и кадров из видео.

Это единственный шаг, требующий ручного действия пользователя перед первым
запуском новой функциональности. Инструкция добавляется в `ПЕРЕД-СТАРТОМ.md`.

## Обработка ошибок

Текущий `ingest_directory` (`kf/ingest.py`) не ловит исключения на уровне
отдельного файла — сбой `extract_text()` на одном файле обрывает весь прогон
индексации. С текстовыми форматами это почти никогда не проявлялось, но
OCR/whisper/сетевой vision-вызов кратно повышают шанс сбоя на конкретном файле
(битый файл, нет сети, неподдерживаемый кодек).

Изменение: вызов `extract_text()` в цикле `ingest_directory` оборачивается в
try/except — при ошибке разбора конкретного файла пишется предупреждение (путь
+ причина), обработка продолжается со следующего файла. `IngestStats` получает
новое поле `files_failed`, выводится в конце прогона.

Если не установлен Tesseract или ffmpeg — при первой попытке разбора
картинки/видео должна быть понятная ошибка с инструкцией «установи X», а не
сырой traceback.

## Тестирование

По уже принятой в проекте дисциплине (TDD, реальные интеграционные тесты без
моков там, где это бесплатно и осмысленно — см. прецедент: `call_llm`/`ask` в
принципе не покрыты автотестами, проверяются вручную живым прогоном):

- **`kf/ocr.py`** — реальный тест: сгенерировать тестовую картинку с известным
  текстом через `Pillow` (аналогично тому, как `fpdf2` уже используется в
  dev-зависимостях для тестовых PDF) → прогнать через Tesseract → проверить
  распознанный текст.
- **`kf/video.py`** — реальный тест на маленькой видео-фикстуре (несколько
  секунд, сгенерировать через ffmpeg в самом тесте или положить в
  `tests/fixtures/`) → проверить, что аудио и кадры извлекаются.
- **`kf/transcribe.py`** — реальный тест с маленькой аудио-фикстурой, модель
  `small` скачивается один раз и кэшируется в `MODEL_CACHE_DIR` (как fastembed).
- **`kf/vision_caption.py`** — **не мокается и не покрывается автотестом** (по
  прецеденту `call_llm`), сеть+деньги; проверяется вручную живым прогоном.
- **`kf/extract.py`/`kf/ingest.py`** — юнит-тесты на диспетчеризацию по
  расширению и на новую try/except логику (файл с ошибкой не обрывает прогон).

## Вне рамок этой задачи

- Полуавтоматический сценарий (разбор картинки/видео мной в разговоре, без
