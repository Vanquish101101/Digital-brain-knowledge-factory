# Расширение форматов: ссылки, аудио, Excel — дизайн

Дата: 2026-07-18

## Проблема

Пользователь хочет присылать материал в разных форматах с пометкой «добавь в базу знаний» и
быть уверенным, что он реально обрабатывается. Проверка показала расхождение между тем, что
написано в `карта_бз.md` (текст подразумевает поддержку всех форматов), и тем, что реально
умеет `kf/extract.py`:

- **Ссылки** (на статьи, на видео) — не поддерживаются вообще: `kf.py ingest` сканирует
  только локальные файлы в папке, скачивать интернет не умеет.
- **Аудио, голосовые записи** отдельным файлом (не внутри видео) — не поддерживаются:
  транскрипция сейчас зашита только внутри обработки видео.
- **Excel-таблицы** (настоящий `.xlsx`) — не поддерживаются, только `.csv` как обычный текст.

Также обнаружено: даже после добавления поддержки в `kf/extract.py`, файлы с новыми
расширениями не будут подхвачены сканированием, пока их нет в allowlist'е
`kf/scope.py::INCLUDED_EXTENSIONS` — это отдельный, обязательный к правке файл.

## Цели

- Новая команда `kf.py ingest-url <url> --dest <папка>`, скачивающая статью или видео по
  ссылке, сохраняющая текст как файл в указанный раздел vault и сразу индексирующая его через
  обычный `kf.py ingest`-пайплайн (эмбеддинг + LLM-заметка + граф знаний).
- Статьи — через `trafilatura` (локально, бесплатно, без внешних API).
- Видео (YouTube) — сперва `youtube-transcript-api` (готовые субтитры, дёшево), при неудаче —
  `yt-dlp` (скачивание аудио) + уже существующий `transcribe_audio` (Whisper).
- Если `trafilatura` вернула подозрительно скудный текст — команда не проваливается молча, а
  явно предупреждает об этом в выводе, чтобы пользователь/агент мог вручную докрутить через
  более сильный инструмент (см. регламент ниже).
- Аудио и голосовые записи отдельными файлами (`.mp3`, `.wav`, `.ogg`, `.m4a`) —
  транскрибируются тем же `transcribe_audio`, что уже используется для видео.
- Excel-таблицы (`.xlsx`) — извлечение текста из всех листов/ячеек.
- `карта_бз.md` и `CLAUDE.md` приводятся в соответствие с реальными возможностями: без
  приукрашивания там, где формат не поддерживается автоматически, и с описанием ручной
  эскалации на Firecrawl.

## Не цели (явно не делаем в этой итерации)

- Не встраиваем Firecrawl (или любой другой MCP-инструмент) внутрь `kf.py` — архитектурно
  недостижимо: `kf.py` — самостоятельный офлайн-скрипт, не подключён к MCP. Работа с Firecrawl
  остаётся ручным шагом ассистента, а не автоматикой `kf.py`.
- Не переносим ничего из отдельного проекта "Marketing agency Project" (Surya OCR, whisperx,
  PySceneDetect) — это подтверждённый, но отдельный будущий апгрейд, зафиксированный в памяти
  ассистента, стартует только когда тот проект пользователь объявит готовым.
- Не поддерживаем видеохостинги, кроме YouTube, для пути "готовые субтитры" — на прочих
  хостингах сразу используется путь "скачать аудио + Whisper".
- Не строим отдельную папку/базу для списков ссылок в vault — маршрут только через явный
  вызов `kf.py ingest-url`.

## Архитектура

### 1. Новый модуль `kf/web_extract.py`

Функции извлечения без побочных эффектов (аналогично `kf/graph.py`'s чистым
функциям + отдельный тонкий сетевой вызов):

- `extract_article(url: str) -> str` — `httpx`-запрос + `trafilatura.extract()`.
- `is_youtube_url(url: str) -> bool` — проверка хоста (`youtube.com`, `youtu.be`).
- `extract_youtube_transcript(url: str) -> str | None` — через `youtube-transcript-api`;
  возвращает `None`, если субтитров нет (не поднимает исключение — отсутствие субтитров
  ожидаемый, а не ошибочный случай).
- `extract_video_via_download(url: str, settings: Settings) -> str` — `yt-dlp` скачивает
  только аудиодорожку во временный файл, дальше — уже существующий
  `kf.transcribe.transcribe_audio`; временный файл всегда удаляется (`finally`), по тому же
  паттерну, что и `kf/extract.py::_extract_video`.
- `extract_from_url(url: str, settings: Settings) -> tuple[str, bool]` — диспетчер: если
  `is_youtube_url` — сначала `extract_youtube_transcript`, при `None` — fallback на
  `extract_video_via_download`; иначе — `extract_article`. Возвращает `(текст, is_low_quality)`,
  где `is_low_quality` — эвристика по длине текста (порог по аналогии с уже существующим
  `settings.image_caption_threshold_chars`, только на уровне статьи, не отдельного слова).

### 2. Новая команда `kf.py ingest-url <url> --dest <папка>`

В `kf/cli.py`: скачивает контент через `extract_from_url`, сохраняет как markdown-файл в
`raw-data-repository/<dest>/` (имя файла — из заголовка страницы/видео или временной метки,
если заголовок не удалось определить), затем вызывает существующую логику одного файла из
`ingest_directory` (или короткий прямой путь: `record_ingested` + `_store_text` +
синтез-заметка + граф — переиспользуя уже существующие функции `kf/ingest.py`, без
копирования их кода). Если `is_low_quality` — печатает явное предупреждение с рекомендацией
ручной эскалации на Firecrawl.

### 3. Аудио отдельным файлом

В `kf/extract.py`:
- `AUDIO_EXTENSIONS = {".mp3", ".wav", ".ogg", ".m4a"}`
- `_extract_audio(path: Path, settings: Settings) -> str` — прямой вызов
  `transcribe_audio(path, settings.whisper_model_size, settings.model_cache_dir)`, без
  промежуточного шага извлечения звука (файл уже аудио).
- `extract_text()` получает новую ветку по `AUDIO_EXTENSIONS`.

### 4. Excel (.xlsx)

Новая зависимость `openpyxl`. В `kf/extract.py`:
- `_extract_xlsx(path: Path) -> str` — читает все листы, для каждого — все непустые ячейки
  построчно (аналогично уровню детализации `_extract_docx`, без попытки сохранить точную
  табличную структуру — цель поиска по смыслу, не рендеринг таблицы).
- `extract_text()` получает новую ветку по `.xlsx`.

### 5. Обязательное обновление `kf/scope.py`

`INCLUDED_EXTENSIONS` — добавить `.mp3`, `.wav`, `.ogg`, `.m4a`, `.xlsx`. Без этого шага
файлы новых форматов не будут даже сканироваться `ingest_directory`, независимо от того, что
`extract_text()` их уже умеет читать — это два независимых слоя защиты, оба должны знать про
новый формат.

### 6. Документация

- `карта_бз.md`: раздел про форматы получает честное разделение на "поддерживается
  автоматически" (текст/PDF/docx/xlsx/картинки/видео/аудио — включая новые
  `kf.py ingest-url` для ссылок) и "требует моего ручного участия" (сайты, где `trafilatura`
  не справилась — эскалация на Firecrawl).
- `CLAUDE.md`: в «Регламент автодобавления знаний» добавляется шаг про `kf.py ingest-url` для
  ссылок и явный шаг эскалации на Firecrawl при предупреждении о низком качестве извлечения.

## Риски и допущения

- Порог "подозрительно скудного текста" для статей — эвристика (длина текста в символах),
  как и все аналогичные пороги в проекте (`image_caption_threshold_chars`); не идеальна, но
  консистентна со стилем уже принятых решений.
- `yt-dlp` может периодически требовать обновления сигнатур/cookies при изменениях на стороне
  YouTube — та же известная эксплуатационная особенность, что уже документирована как риск для
  видео-пайплайна в целом (не новый риск, а расширение существующего).
- Реальная сетевая часть (`extract_article`, `extract_youtube_transcript`,
  `extract_video_via_download`) не покрывается юнит-тестами — тот же принцип, что уже принят
  для `kf/graph.py`'s LLM-вызова и `kf/embeddings.py`'s OpenRouter-вызова: проверяется вручную
  на реальных данных при финальной верификации фичи.
